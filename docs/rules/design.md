# Design rules

`design` スキルと UI を伴う `feature` が読む。

## デザイン原則が法

- `docs/PROJECT.md` の「デザイン原則」と「トーン」は全 UI 判断の上位ルール。迷ったらここに照らす。
- 世界観はここ(`design.md`)で育てる。新しい美意識の決定・アンチパターンを見つけたら追記する。

## AI感の罠を避ける(NOリスト)

- 意味のないグラデーション乱用、過剰なシャドウ、中央寄せだけの単調レイアウト。
- 全部 `rounded-lg` + `border` の没個性カード。
- リテラル色の散乱(→ トークン化)。
- ※ プロジェクト固有の NO は onboarding/design で `docs/PROJECT.md` に蓄積する。

## アクセシビリティ

- コントラスト比 WCAG AA 以上。
- インタラクティブ要素は role/label を持ち、キーボード操作可能。
- shadcn(Base UI)のプリミティブを土台にし、a11y を自前で壊さない。

## トークン

- 色・タイポ・間隔は `globals.css` の `@theme` と shadcn トークンを正典とする。
- 新トークンを足したら名前と用途をここに1行記す。

## pencil / 画面設計

- 画面モックを作るなら pencil(`.pen`)を使い、`docs/design/NNNN-*.md` から参照する。
- 未統合コンポーネントは `.pen` をビジュアル仕様とし、統合時に再検証する。
