## 一言で

型安全フルスタック(Next.js + Hono + Drizzle + Better Auth)の最小スターター ── ログインと、ログイン後のダッシュボードがすぐ動く土台。

## 何を FEEL させるか

余計なものがない、静かで素直な土台。開いた人が「これなら自分の機能を足せる」と即座に見通せる軽さ。

## ターゲット

- ペルソナ: このテンプレを clone して自分のアプリを作り始める開発者(個人 / 小チーム)
- 主な利用文脈: 新規プロダクトの初日。認証とダッシュボードの骨組みを書かずに済ませ、本題の機能から始めたい

## アンチゴール(このプロダクトでやらないこと)

- 特定プロダクトの業務ロジックを盛らない ── 土台に留める。機能は clone した先で足す。
- 認証の作り込みをしない ── まずは Email + パスワードの1経路のみ。OAuth / MFA / SSO は必要になった人が足す。
- ダッシュボードを作り込まない ── ログイン後に「入れた」と分かる最小画面。凝った可視化やウィジェットは置かない。

## デザイン原則(3〜5個)

1. **素直 > 巧み** ── 初見の開発者が構造を追える。隠れた魔法を作らない。
2. **最小デモ** ── 各画面は「動くことを示す最小限」。装飾より見通し。
3. **中立なトーン** ── 特定ブランド色を持たせない。clone した人が上書きしやすい余白を残す。
4. **型安全を体現** ── DB→API→UI が型でつながる様を、最小コードで示す。

これらは全 UI 判断の上位ルール(`docs/rules/design.md` から参照)。

## トーン

- 言葉づかい: 簡潔・説明的(テンプレなので「読めば分かる」を優先)
- ビジュアル: モノクロ基調 / 余白多め / 高コントラスト。装飾は最小。

## 主要トークン(任意・分かる範囲で)

- 基調色: ニュートラル(white / neutral-900)
- アクセント: 1色のみ(後で clone 先が差し替えやすいよう控えめに)
- フォント: システム / Next 既定
- 角丸 / 影の方針: 角丸は控えめ、影は浅く(フラット寄り)
